Agent

Fikrlovchi yordamchi tizim

LLM + xotira + ruxsat berilgan asboblar bir tizimda. Vazifani tushunadi, mosini tanlaydi, natijani izohlaydi va belgilangan chegaradan chiqsa — inson xodimiga uzatadi. Mustaqil qaror egasi emas: huquq, limit va javobgarlik bank siyosati bilan belgilanadi.

Tool Use

Asbobdan foydalanish

AI ruxsatli asbobni chaqiradi. Bankda ikki guruh: read tool (o'qish) — balans, hujjat, kursni oladi; write tool (yozish) — tizimga ta'sir qiladi (xat, hisob, fayl) va odatda inson tasdig'i, limit va auditni talab qiladi.

Audit

Bitta agent — qachon yetarli, qachon multi-agent kerak.

Internet "100 ta agent birga ishlaydi" deb yozadi. Bankda — bittasi, sodda, aniq agent yaxshiroq.

✓ Bitta agent — yaxshi
  • Vazifa aniq, bir o'qda — savol → tool → javob
  • Mas'uliyat 1 bo'limga tushadi (operator, KYC)
  • Tool soni 3–6 ga kiradi
  • Sikl natijasi tekshiriladigan
  • Audit log bitta joyda yig'iladi
✗ Bitta agent — yomon
  • Vazifa 5+ bo'limga tarqalgan (kredit + muvofiqlik + xazina)
  • Har bo'limning o'z qoidasi — bitta agent bilolmaydi
  • Tool soni 15+ — agent yo'qolib qoladi
  • Ikki tomonlama qaror (mijoz + bank manfaati)
  • Birinchi xato butun jarayonni to'xtatadi
